# Support & Community

Need help with Michelangelo AI? Connect with the core team and developer community through our dedicated channels.

## Get Help

- **[GitHub Discussions](https://github.com/michelangelo-ai/michelangelo/discussions):** Our primary channel for general Q&A, troubleshooting, technical support, and sharing ideas. Search existing discussions before opening a new one — your question may already be answered.

- **[Slack Workspace](https://michelangelo-ai.slack.com/):** For real-time chat, networking, and focused conversations with other developers in the ecosystem.

- **[Email](mailto:michelangelo-oss@uber.com):** For private, sensitive, or 1-on-1 inquiries that cannot be discussed publicly (for example, security disclosures or enterprise partnerships).

## Report a Bug

Found something broken? [Open a GitHub issue](https://github.com/michelangelo-ai/michelangelo/issues/new/choose) using the bug report template. Include reproduction steps, expected vs. actual behavior, and your environment details.

## Request a Feature

Have an idea for a new feature? Start with [GitHub Discussions](https://github.com/michelangelo-ai/michelangelo/discussions) to gauge interest. For major architectural changes, see the [enhancements repository](https://github.com/michelangelo-ai/enhancements) for the RFC process.

## Contribute

Want to contribute code, docs, or examples? See the [Contributing Guide](https://github.com/michelangelo-ai/michelangelo/blob/main/CONTRIBUTING.md) for how to get started.

## Security Issues

**Do not** report security vulnerabilities through public GitHub issues. Instead, submit them through Uber's [HackerOne](https://hackerone.com/uber) program.
